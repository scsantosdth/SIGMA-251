## **📋 CONTEXTO GENERAL DEL PROYECTO**

**Título:** Desarrollo de prototipo para medición de variables ambientales de sistemas agroindustriales con visualización en dispositivos remotos.

**Autores:** Brayan Julian Niño Hurtado & Sergio Camilo Santos Uribe

## **🎯 OBJETIVOS CLAVE**

### **Objetivo General:**
Desarrollar un prototipo de sistema IoT para medición de variables ambientales en sistemas agroindustriales, con almacenamiento local y sincronización dinámica en base de datos en la nube.

### **Objetivos Específicos:**
1. ✅ Diseñar arquitectura IoT con selección de sensores adecuados
2. ⚠️ Desarrollar firmware/software de adquisición de datos
3. 🔴 Implementar mecanismo de almacenamiento híbrido (local + nube)
4. ⚠️ Diseñar interfaz de visualización remota
5. 🔴 Evaluar desempeño en condiciones reales

### **Variables a medir:**
- Temperatura ambiental (°C)
- Humedad del suelo (%) 
- Radiación solar (W/m²)
- Nivel de almacenamiento local (% capacidad)

## **🔗 ARQUITECTURA ACTUAL**

```
Waspmote (sensores) 
    ↓ (XBee wireless)
Puerto Serial COM8 (PC)
    ↓
Script Python (serial_reader.py)
    ↓ (HTTP POST)
FastAPI (localhost:8000)
    ↓ (SQLAlchemy)
PostgreSQL Database
    ↓ (REST API)
React Frontend (localhost:3000)
    ↓ (Fetch API)
Usuario en navegador
```

### **Endpoints principales:**
- `POST /api/mediciones/waspmote` - Recibir datos de sensores
- `POST /api/estado-sistema/waspmote` - Estado de batería
- `GET /api/mediciones/waspmote/latest` - Última medición
- `GET /api/mediciones/waspmote/historical` - Datos históricos

## **🎮 PUNTOS DE CONFUSIÓN ACLARADOS**

### **1. "Almacenamiento local en dispositivo móvil":**
- **Plan original:** Sugiere app móvil que almacena datos offline
- **Implementación actual:** Script Python guarda en `sensor_backup.txt` en PC
- **Decisión tomada:** Implementaremos almacenamiento offline en el NAVEGADOR web (IndexedDB) para cumplir el espíritu del proyecto

### **2. "Visualización en dispositivos remotos":**
- **Estado actual:** Solo funciona en `localhost` de la PC
- **Requerimiento:** Sistema debe ser accesible vía internet
- **Solución pendiente:** Despliegue en servidor cloud o configuración para LAN

### **3. Responsive vs App nativa:**
- **Confirmado:** Se desarrolla aplicación WEB (React), no app nativa
- **Responsive:** La web debe verse bien en PC y móviles
- **Offline:** Navegador almacenará datos cuando no haya conexión
